針對 **`basic-04-unit-h-bridge` (H 橋驅動原理：解鎖馬達控制的鑰匙)** 單元，這是學員從「小 LED 實驗」轉向「大動力控制」的物理門檻。它要求學員具備 **功率驅動邏輯 (Power Driving Logic)** 的思維，學習如何透過「控流矩陣 (H-Bridge Map)」實現正反轉，並理解物理世界的 **「慣性 (Inertia)」** 與 **「動態制動 (Dynamic Braking)」** 對控制精度的影響。

以下是在 **GitHub Classroom** 部署其作業 (Assignments) 的具體建議：

### 1. 範本倉庫 (Template Repo) 配置建議
H 橋作業的核心在於「邏輯對應」與「API 封裝」，範本應包含完整的軟硬體接口定義，建議包含：
*   **📂 `src/main.cpp`**：**核心 API 封裝區**。學員需在此實作一個專業的 `drive(int speed)` 函式，處理 `-255` (全速倒車) 到 `255` (全速前進) 的邏輯映射。
*   **📂 `include/motor_pins.h`**：引腳映射中心。預先定義 AIN1, AIN2, STBY (TB6612 腳位)，學員需學習如何透過 STBY 引腳進行「全局凍結」。
*   **📂 `README.md`**：**驅動真值表實測報表**。提供表格讓學員記錄：「IN1, IN2 輸入」 vs 「馬達實際轉動行為」。
*   **📂 `docs/BRAKE_TEST.md`**：**動態阻尼實驗室**。要求學員記錄：在全速轉動時，切換至「急煞 (Brake)」與「空轉 (Coast)」模式下，馬達停止旋轉所需的時間或慣性滑動距離。

### 2. 作業任務部署細節

#### 任務 1：「邏輯審計」驅動真值表實測 (Logic Audit Lab)
*   **目標**：透過實測掌握 H 橋的四種基本組態，建立「控制位元」與「物理動作」的直覺。
*   **Classroom 部署建議**：
    *   **遍歷真值表**：要求學員撰寫 code 遍歷 (H, L), (L, H), (H, H), (L, L) 四種組合。
    *   **記錄動作**：在 README 中彙整每種組合對應的馬達動作（正轉、反轉、煞車、空轉）。
    *   **驗證要點**：README 中必須清楚標註 (H, H) 產生的物理現象，探討為何這被稱為「短路煞車」。

#### 任務 2：「動態阻尼」急煞 vs. 緩停實驗 (Dynamic Damping Lab)
*   **目標**：理解反向電動勢 (Back-EMF) 產生的制動力，這對未來「精準停靠」任務至關重要。
*   **Classroom 部署建議**：
    *   **慣性量化量測**：設定馬達全速轉動 2 秒後突然停止。A 組使用 Coast (IN1/IN2 全 LOW)，B 組使用 Brake (IN1/IN2 全 HIGH)。
    *   **分析現象**：觀察並描述兩者「停止瞬間」的差異。
    *   **驗證要點**：在報告中解釋：為何 Brake 模式能有效縮短停止後的滑行距離？（引用電磁感應原理）。

#### 任務 3：「動力總控」雙向動力 API 封裝 (Omni-Control API Lab)
*   **目標**：將複雜的 H 橋引腳操作抽象化為單一 API，建立具備保護邏輯的專業動力接口。
*   **Classroom 部署建議**：
    *   **驅動 API 實作**：實作 `drive(int speed)`，正數表示前進，負數表示後退，`0` 執行 Brake 邏輯。
    *   **邊界條件處理**：處理當輸入值大於 255 或小於 -255 的限制保護 (Constrain)。
    *   **序列上演習**：撰寫一個 Sequencer，執行：全速前進 2s -> 急煞 0.5s -> 半速後退 1s -> 空轉。
    *   **最終 Commit**：封裝完整的馬達控制類別並 Push。

### 3. 馬達驅動專家導師點評標準 (Motion Control & Protection Standards)
此單元的價值在於 **「對硬體邏輯的精確映射與代碼架構的簡潔度」**：
*   [ ] **API 封裝質量**：`drive()` 函式是否能優雅地處理正負號方向的轉碼邏輯？
*   [ ] **保護邏輯意識**：是否包含 STBY (Standby) 引腳的控制？是否有處理 Shoot-through (直通短路) 的預防思維？
*   [ ] **物理洞察力**：實驗日誌是否能分辨「空轉」與「煞車」在不同場景（如避障 vs 節能）的應用價值？
